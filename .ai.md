# AI Context for PY_PC_power_analyzer

## Project Purpose
PC power consumption analysis and optimization for an AMD Ryzen 9 5900X / RX 6900 XT system.
Ground truth power measurement via NETIO 4KF smart power socket at 192.168.178.118.

## Architecture
- **PowerSaver.ps1** / **GamingMode.ps1**: PowerShell scripts (require admin) that toggle between power-saving and max-performance modes
- **ZenControl/**: C# .NET 8 console app that sends commands directly to the AMD SMU (System Management Unit) via ZenStates-Core library + WinRing0 kernel driver
- **read_sensors.py**: Python script using LibreHardwareMonitor via pythonnet to read hardware sensors

## Key Technical Details
- CPU: AMD Ryzen 9 5900X, Vermeer, Family 19h Model 0x21, 2 CCDs × 6 cores, AM4
- SMU RSMU command IDs: PPT=0x53, TDC=0x54, EDC=0x55, HTC=0x56 (Zen3/Zen2Settings)
- ZenStates-Core v1.75 (last WinRing0 version, before PawnIO migration)
- NETIO API: `GET http://192.168.178.118/netio.json` with Basic auth `netio:netio`
- Output 1 ("PC RIG"): `.Outputs[0].Load` = watts from wall
- Windows German locale, Hyper-V enabled, Python via `py` launcher

## Build Instructions
```bash
# ZenControl
cd ZenControl
dotnet build -c Release
# Deps already in bin/Release/net8.0-windows/: inpoutx64.dll, WinRing0x64.dll, WinRing0x64.sys
```

## Admin Elevation Pattern
Scripts use `#Requires -RunAsAdministrator`. For non-interactive admin execution:
```powershell
$bat = "@echo off`ncd /d ""$dir""`nZenControl.exe info > ""%TEMP%\out.txt"" 2>&1"
$bat | Set-Content "$env:TEMP\run.bat" -Encoding ASCII
Start-Process "$env:TEMP\run.bat" -Verb RunAs -Wait -WindowStyle Hidden
```

## Conventions
- All scripts expect NETIO at 192.168.178.118 with Basic auth netio:netio
- ZenControl requires admin (WinRing0 kernel driver)
- Core disable fuse maps are hardware-burned — cannot be changed via software
- RyzenAdj does NOT support desktop Zen 3 Vermeer
